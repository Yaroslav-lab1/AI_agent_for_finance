# Легенда 4: Frontend и пользовательский интерфейс

## Роль

Участник отвечал за React-интерфейс приложения: авторизацию, dashboard, формы операций, личный кабинет, AI-импорт, предпросмотр кандидатов, статистику и визуальное оформление.

## Что сделал

- Реализовал страницу входа и регистрации с полями имени, логина, email, пароля и повторного ввода пароля.
- Добавил панель профиля, где пользователь может менять имя, логин, email и пароль.
- Показал на сайте требование: для изменения любых данных аккаунта нужен текущий пароль.
- Реализовал форму ручного добавления операции.
- Реализовал AI-импорт в трех режимах: текст, CSV и изображение.
- Сделал file dropzone для CSV и изображений, чтобы загрузка файла выглядела единообразно с текстовым режимом.
- Добавил таблицу предпросмотра AI-кандидатов с возможностью редактировать сумму, тип, категорию, дату и комментарий перед подтверждением.
- Реализовал список операций с фильтрами по датам, типу, категории и поиску.
- Реализовал сортировку операций кликом по заголовкам таблицы: дата, тип, сумма, категория, комментарий и источник.
- Установил сортировку по умолчанию по дате по убыванию.
- Добавил две кольцевые диаграммы расходов: за все время и за выбранный месяц.
- Добавил выбор месяца для второй диаграммы.
- Добавил единый legend цветов категорий и баланс в центре каждой диаграммы.
- Поправил адаптивную верстку, вертикальные отступы между блоками, расположение кнопок и перенос длинных комментариев.

## Что может рассказать на защите

Frontend построен вокруг контроля пользователя. AI только предлагает операции, а интерфейс дает возможность проверить, исправить и подтвердить результат. Dashboard показывает как список операций, так и визуальную статистику, поэтому пользователь быстро понимает общую картину финансов.

Можно отдельно показать два ключевых сценария: ручное добавление операции и AI-импорт с предпросмотром кандидатов.

## Возможный вопрос на защите

**Почему AI-кандидаты редактируются до подтверждения?**

Потому что пользователь должен контролировать финансовые записи. Даже хорошая LLM может ошибиться в категории, дате или комментарии, поэтому интерфейс показывает промежуточный результат и дает исправить его до записи в историю операций.
