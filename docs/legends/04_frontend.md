# Легенда 4: Frontend и пользовательский интерфейс

## Роль

Участник отвечал за React-интерфейс приложения: авторизацию, dashboard, формы операций, AI-импорт, предпросмотр кандидатов, статистику и визуальное оформление.

## Что сделал

- Реализовал страницу входа и регистрации с полями имени, логина, email, пароля и повторного ввода пароля.
- Добавил панель профиля, где пользователь может менять имя, логин, email и пароль.
- Показал на сайте требование: для изменения любых данных аккаунта нужен текущий пароль.
- Реализовал форму ручного добавления операции.
- Реализовал AI-импорт в трех режимах: текст, CSV и изображение.
- Сделал удобные file dropzone для CSV и изображений.
- Добавил таблицу предпросмотра AI-кандидатов с возможностью редактировать поля перед подтверждением.
- Реализовал список операций с фильтрами и сортировкой кликом по заголовкам таблицы.
- Добавил две кольцевые диаграммы расходов: за все время и за выбранный месяц.
- Добавил единый legend цветов категорий и баланс в центре каждой диаграммы.
- Поправил адаптивную верстку, отступы между блоками и перенос длинных комментариев.

## Что может рассказать на защите

Frontend построен вокруг контроля пользователя. AI только предлагает операции, а интерфейс дает возможность проверить, исправить и подтвердить результат. Dashboard показывает как список операций, так и визуальную статистику, поэтому пользователь быстро понимает общую картину финансов.
