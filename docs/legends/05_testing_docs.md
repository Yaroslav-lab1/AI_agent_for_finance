# Легенда 5: Тестирование, документация и подготовка к защите

## Роль

Участник отвечал за проверку качества проекта, документацию, QA-материалы и подготовку объяснения архитектуры для защиты.

## Что сделал

- Поддерживал README в актуальном состоянии: запуск, переменные окружения, API, AI-импорт, OCR и сценарий демонстрации.
- Подготовил QA-документ с ответами на типовые вопросы по проекту.
- Описал обоснование технологий: backend, frontend, БД, OCR, LLM, Docker и тестирование.
- Подготовил легенды участников проекта.
- Проверял, что документация совпадает с текущими таблицами БД, API-эндпоинтами и интерфейсом.
- Поддерживал backend-тесты для авторизации, профиля, операций, статистики и импортов.
- Проверял frontend-сборку после крупных изменений интерфейса.

## Что может рассказать на защите

Качество проекта обеспечивается не только тестами, но и понятной архитектурной документацией. Документы помогают объяснить, почему AI-импорт сделан через кандидатов, зачем нужен OCR, почему используется PostgreSQL и как пользователь контролирует финальную запись операций.
