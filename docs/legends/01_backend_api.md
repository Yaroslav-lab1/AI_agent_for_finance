# Легенда 1: Backend API и доменная модель

## Роль

Участник отвечал за backend-слой приложения: REST API, модели данных, бизнес-логику транзакций и связь с базой.

## Что сделал

- Спроектировал основные сущности: пользователь, категория, транзакция, импорт, кандидат импорта, audit log.
- Реализовал FastAPI-роуты для авторизации, транзакций, категорий, статистики и импортов.
- Настроил dependency для текущего пользователя и изоляцию данных по `user_id`.
- Добавил сервисный слой, чтобы роуты не содержали всю бизнес-логику.

## Что может рассказать на защите

Главный акцент: backend не просто принимает данные, а разделяет черновые кандидаты импорта и подтверждённые транзакции. Это важно для финансового приложения, потому что AI может ошибаться, а пользователь должен сохранять контроль.

## Какие файлы знает

- `backend/app/api/routers/*.py`
- `backend/app/services/*.py`
- `backend/app/models/*.py`
- `backend/app/schemas/*.py`

Проверял сценарии CRUD, авторизации, user scope и статистики через API-тесты.
