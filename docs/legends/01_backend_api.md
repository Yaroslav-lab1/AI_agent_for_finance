# Легенда 1: Backend API и доменная логика

## Роль

Участник отвечал за backend-слой проекта: REST API, бизнес-логику операций, авторизацию, профиль пользователя и связь с базой данных.

## Что сделал

- Спроектировал основные сущности backend: `User`, `Category`, `Transaction`, `ImportJob`, `ImportCandidate`, `AgentAuditLog`.
- Реализовал FastAPI-роуты для регистрации, входа, текущего пользователя и обновления профиля.
- Добавил правило безопасности: для изменения имени, логина, email или пароля пользователь должен указать актуальный пароль.
- Реализовал CRUD операций: создание вручную, список с фильтрами, обновление и удаление.
- Реализовал API статистики: общий баланс, доходы, расходы и расходы по категориям с фильтром по периоду.
- Настроил user scope: пользователь видит и меняет только свои операции и импорты.
- Разделил роуты, сервисы и репозитории, чтобы бизнес-логика не была смешана с HTTP-слоем.

## Что может рассказать на защите

Главная идея backend: финальные финансовые записи и AI-кандидаты разделены. AI может ошибаться, поэтому backend сначала сохраняет кандидатов импорта, а пользователь подтверждает только нужные операции. Такой подход безопаснее для финансового приложения.
