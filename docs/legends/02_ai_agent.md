# Легенда 2: AI-агент и импорт операций

## Роль

Участник отвечал за AI-часть проекта: пайплайн разбора банковских уведомлений, поддержку текста, CSV и изображений, structured output, нормализацию результата и проверку качества найденных операций.

## Что сделал

- Реализовал `AgentPipeline`, который объединяет все этапы AI-импорта.
- Добавил `TextInputTool` для нормализации текстовых уведомлений.
- Добавил `CsvInputTool` для чтения CSV-файлов, определения разделителя и приведения строк к единому текстовому формату.
- Добавил `ImageOcrTool` для обработки PNG/JPEG/WebP через Pillow, pytesseract и Tesseract OCR.
- Подключил LLM-клиенты: `MockLLMClient`, `OllamaLLMClient`, `OpenAICompatibleLLMClient`.
- Настроил JSON Schema для structured output, чтобы модель возвращала машинно читаемый список операций.
- Добавил prompt-инструкции для банковских уведомлений: игнорировать баланс, номера карт, технические идентификаторы и использовать дату по умолчанию, если в уведомлении есть только время.
- Реализовал нормализацию суммы и даты после LLM.
- Добавил определение типа операции, классификацию категории и проверку дублей.
- Обеспечил сохранение результатов импорта как кандидатов, а не финальных операций.
- Сохранял ход работы агента в `agent_audit_logs`.
- Провел базовые замеры производительности реального `qwen3.5:9b`: 1 операция около 3-4 секунд, 10 операций около 20 секунд.

## Что может рассказать на защите

AI-агент работает не как "магическая запись в базу", а как контролируемый пайплайн. Разные источники сначала приводятся к единому тексту, затем LLM возвращает JSON по схеме, а backend дополнительно нормализует и проверяет результат. Пользователь остается финальной точкой контроля.

Важно подчеркнуть, что текущий `qwen3.5:9b` работает как текстовая LLM. Поэтому изображения сначала проходят OCR, а уже потом обрабатываются как текст.

## Ответственность за качество

Проверял, что текст, CSV и OCR-текст проходят через общий пайплайн, что LLM-ответ валидируется, а ошибочные или подозрительные результаты не попадают в финальные операции без подтверждения пользователя.

## Возможный вопрос на защите

**Почему используется structured output?**

Потому что backend должен получить предсказуемую структуру: сумму, дату, тип, категорию, комментарий и confidence. JSON Schema уменьшает риск получить свободный текст, который сложно надежно распарсить.
